### Vite とは

- 多くの機能を備えたツール

    - 人によってはフロントエンドツールだったり、[ビルドツール](#ビルドツール)や[モジュールバンドラー](#モジュールバンドラー)だったりと形容の仕方は様々

        - 個人的にはビルドツールという説明がしっくりくる
    
    <br>

    - 主に以下の機能を利用できる

        - 開発サーバー (ローカル環境でサーバーをたてることができる)

        <br>

        - モジュールバンドラー

            - ★内部的には [Rollup](https://zenn.dev/lidqqq/articles/1ed2ca8a1bb41b3d002d) というモジュールバンドラーを使っている

                - ★★常に Rollup を使っているわけではなく、開発ビルドモードでは esbuild、本番ビルドモードでは Rollup を内部的に使っているらしい

        <br>

        - TypeScript のサポート

            - Vite はネィティブで TypeScript をサポートしている

                - ★内部的には esbuild を使ってトランスパイルしている

            - webpack はネイティブで TypeScript をサポートしていないので、webpack で TypeScript を使いたい場合は loader を別途インストールする必要が出てくる

        <br>

        - CSS 処理

            - sass, scss を css にコンパイル

        <br>

        - プラグインの利用でさまざまな機能を追加できる

            - Vue の 単一ファイルコンポーネント (.vue) ファイルをサポートするプラグインである `@vitejs/plugin-vue` などがある


<br>
<br>

### モジュールバンドラー

- 複数の js ファイル間の依存関係を解決し、まとめるツール

### ビルドツール

- そもそも、フロントエンド開発での「ビルドツール」って何?

    - 以下の処理を行うツールのこと

        - ts ファイルや sass/scss ファイルのコンパイル

        - ファイルのバンドル (=[モジュールバンドラー](#モジュールバンドラー))

            - モジュールバンドラーはビルドツールの**サブセット**と捉えることも可能

        - 静的ファイルの圧縮

        - 環境変数の注入

            など
